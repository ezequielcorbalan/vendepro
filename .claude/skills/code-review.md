# Skill: Code Review

## Priority Order
1. Correctness — does it do what it should?
2. Types — proper TypeScript, no unsafe casts beyond necessary
3. UX — loading/empty/error states, responsive
4. Regressions — does it break existing features?
5. Maintainability — readability, naming, structure

## Format
- Flag production risks clearly
- Concrete feedback, not vague suggestions
- If suggesting a change, show the code
- Separate "must fix" from "nice to have"
