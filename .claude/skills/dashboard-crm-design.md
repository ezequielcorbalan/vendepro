# Skill: CRM Dashboard Design

## Principles
- Utility first — every element must serve a purpose
- Show: filters, quick actions, metadata, ownership, urgency, next steps
- No giant whitespace — use information density wisely
- Mobile: cards/lists that are actually usable
- Desktop: leverage width for grids, tables, side panels

## Card Design
- Always show: entity name, status badge, key metric, responsible agent
- Optional: phone link, WhatsApp link, next step date, overdue indicator
- Actions: inline or on hover/expand

## Kanban Design
- Columns = pipeline stages
- Cards = entities in that stage
- Must have: count per stage, quick advance button, SLA indicator
